### Modular Architecture

**Key Principles**:
1. **Separation of Concerns** - Each module has a single responsibility
2. **Loose Coupling** - Modules communicate via events and APIs
3. **High Cohesion** - Related functionality grouped together
4. **Hot Swapping** - Modules can be loaded/unloaded without restart
5. **License-based** - Modules enabled based on customer license

### Event-Driven Architecture

**Event Flow**:

**Key Benefits**:
- **Scalability**: Add more consumers to handle load
- **Resilience**: Failed consumers don't affect others
- **Auditability**: Complete event log
- **Real-time**: Instant propagation of changes

### Microservices Architecture

**Services**:
- **api** - Main API (NestJS)
- **web** - Web application (Next.js)
- **admin** - Admin panel (Next.js)
- **mobile-api** - Mobile-specific API
- **auth** - Authentication service
- **notification** - Notification service
- **document** - Document service
- **engines** - Python computational engines
- **agents** - AI agent services

**Communication**:
- **Synchronous**: REST API, GraphQL
- **Asynchronous**: Kafka events
- **Real-time**: WebSocket

### Database Architecture

**40 Specialized Databases**:
- Each module can have its own database
- Shared schemas for common data
- Read replicas for analytics
- Write-ahead logging for audit

**Example Databases**:
- `soriano_auth` - Users, roles, sessions
- `soriano_policies` - Insurance policies
- `soriano_claims` - Claims management
- `soriano_finance` - Financial data
- `soriano_analytics` - Aggregated analytics

### Scalability Strategy

**Horizontal Scaling**:
- **API**: Scale to N instances behind load balancer
- **Workers**: Scale Kafka consumers independently
- **Databases**: Read replicas for queries
- **Cache**: Redis cluster mode
- **Storage**: Distributed object storage

**Vertical Scaling**:
- **Database**: Upgrade instance size
- **Cache**: Increase memory
- **API**: More CPU/RAM per instance

**Auto-scaling Triggers**:
- CPU > 70% for 5 minutes
- Memory > 75% for 5 minutes
- Request queue > 1000
- Response time > 500ms (P95)
